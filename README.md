# LLM-Based Cluster Validation

An resource that walks through building, evaluating, optimizing, and fine-tuning a [DSPy](https://dspy.ai) program for cluster validation.

The concrete task is **intruder detection**: given six keywords where five belong to one semantic cluster and one is from a different cluster, an LLM must identify the intruder. This task is a vehicle for explaining DSPy concepts — the end goal is fine-tuning `ministral-3:3b` using traces generated by Claude Sonnet as teacher.

---

## What is the pipeline?

```
build_dataset.py          → raw_examples.json
  └─ evaluate.py          → baseline score   (MLflow)
       └─ optimize.py     → BootstrapFewShot (MLflow)
       └─ optimize_gepa.py    → GEPA         (MLflow)
       └─ optimize_finetune.py → BootstrapFinetune — primary goal (MLflow)
            └─ deploy_mlflow.py → serve via MLflow
```

**Optimization order:** BootstrapFewShot → GEPA → BootstrapFinetune.
The first two establish a performance baseline and produce labeled traces; fine-tuning is the primary objective.

---

## Models

| Role    | Model              | Provider              | Used by                  |
|---------|--------------------|-----------------------|--------------------------|
| Student | `ministral-3:3b`   | SGLang (local)        | all scripts (task LM)    |
| Teacher | `claude-sonnet-4-6`| Anthropic API         | `optimize_finetune.py`   |

The teacher generates high-quality (prompt, completion) traces; the student's weights are then updated on those traces via `BootstrapFinetune`.

## Setup

### Prerequisites

- Python 3.12
- [uv](https://docs.astral.sh/uv/) — for dependency management
- [SGLang](https://github.com/sgl-project/sglang) — serving `Ministral-4b-instruct` locally on GPU
- An Anthropic API key (only needed for `optimize_finetune.py`)

### Install

```bash
uv sync
```

### Environment variables

```bash
cp .env.example .env
# Edit .env and set your Anthropic API key:
# ANTHROPIC_API_KEY=sk-ant-...
```

### Start the SGLang inference server

The pipeline scripts expect `Ministral-4b-instruct` to be available at `http://localhost:30000/v1`.
Start it before running any pipeline script:

```bash
python -m sglang.launch_server \
    --model-path mistralai/Ministral-4B-Instruct-v0.3 \
    --port 30000
```

> **Note — GCC version:** SGLang JIT-compiles FlashInfer CUDA kernels at first launch. CUDA requires GCC ≤ 12 as the host compiler. If your system GCC is newer (check with `gcc --version`), compilation will fail with an "unsupported GNU version" error. Fix it by installing GCC 12 and registering it as the system default via `update-alternatives` (the `CUDAHOSTCXX` env var is not sufficient — FlashInfer's ninja build does not pass it through to `nvcc`):
> ```bash
> sudo apt install gcc-12 g++-12
> sudo update-alternatives --install /usr/bin/gcc gcc /usr/bin/gcc-12 12
> sudo update-alternatives --install /usr/bin/g++ g++ /usr/bin/g++-12 12
> ```

## Running via command line

### 1. Generate the dataset

Streams Dutch news articles, runs BERTopic to extract topics, then builds intruder-detection examples. The HuggingFace download is disk-cached — subsequent runs with the same arguments return instantly.

```bash
# Default: 5,000 Dutch articles, 4 examples per topic
python pipeline/build_dataset.py

# Custom volume
python pipeline/build_dataset.py --max-docs 10000 --examples-per-topic 4

# Skip BERTopic and re-generate examples from an existing topics.json
python pipeline/build_dataset.py --skip-bertopic
```

### 2. Evaluate the baseline

```bash
python pipeline/evaluate.py
```

Results are printed to stdout, saved to `outputs/eval_results.json`, and logged to MLflow experiment `cluster-validator-eval`.

### 3. Optimize — BootstrapFewShot

```bash
python pipeline/optimize.py
```

Saves the compiled program to `outputs/program_optimized.json`.

### 4. Optimize — GEPA

GEPA performs reflective prompt optimization: it iteratively proposes better instructions by feeding failures to a reflection LM. The reflection LM is read from the `teacher` block in `config/dspy_config.yaml` (currently `claude-sonnet-4-6`). Requires `ANTHROPIC_API_KEY`.

```bash
python pipeline/optimize_gepa.py
```

Saves the compiled program to `outputs/program_gepa.json`.

### 5. Optimize — BootstrapFinetune (primary goal)

Uses Claude Sonnet as teacher to generate training traces, then fine-tunes `ministral-3:3b`.
Requires `ANTHROPIC_API_KEY` to be set.

```bash
python pipeline/optimize_finetune.py

# Optional flags
python pipeline/optimize_finetune.py \
    --output-dir ./finetuned_model \
    --epochs 2 \
    --use-peft
```

Saves fine-tuned weights to `./finetuned_model/` and the compiled DSPy program to `outputs/program_finetune.json`.

### 6. Deploy with MLflow

```bash
python pipeline/deploy_mlflow.py --output-dir ./finetuned_model

# Then serve the logged model (use the run ID printed by the script):
mlflow models serve -m "runs:/<run_id>/dspy_program" -p 6000
```

## Running via notebook

Open `demo.ipynb` in the root of the repository. The notebook covers:

1. **Setup** — configuring DSPy and loading the dataset
2. **Evaluate** — running `dspy.Evaluate` with MLflow tracking
3. **BootstrapFewShot** — prompt-level optimization
4. **GEPA** — reflective instruction optimization
5. **BootstrapFinetune** — teacher-to-student fine-tuning

Each section includes an explanation of what DSPy is doing under the hood.

```bash
uv run jupyter notebook demo.ipynb
```

## Running with Docker

Build and run the full pipeline container (requires NVIDIA GPU):

```bash
docker build -t cluster-validator .

# Run any pipeline script
docker run --gpus all \
    -e ANTHROPIC_API_KEY=$ANTHROPIC_API_KEY \
    -v $(pwd)/outputs:/app/outputs \
    cluster-validator \
    python pipeline/evaluate.py
```

Ports `30000` (SGLang) and `5000` (MLflow serve) are exposed.

---

## MLflow tracking

Every evaluation and optimization run is automatically logged via `mlflow.dspy.autolog()`.

```bash
uv run mlflow ui
# Open http://localhost:5000
```

Key experiments:
- `cluster-validator-eval` — baseline evaluation runs
- `cluster-validator-optimize` — BootstrapFewShot and GEPA runs
- `cluster-validator-finetune` — BootstrapFinetune runs

## Project structure

```
cluster_validator/      # Installable package — all reusable logic
  config.py             # configure_dspy(), configure_teacher_lm(), load_config()
  module.py             # IntruderDetectionSignature, ClusterIntruderValidator, find_intruder()
  metrics.py            # intruder_exact_match(), gepa_metric()
  data.py               # build_devset() and dataset splitting utilities

config/
  dspy_config.yaml      # Student (ministral) and teacher (Claude Sonnet) LM settings

data/                   # Input datasets (generated by build_dataset.py)
  topics.json
  raw_examples.json

pipeline/               # Executable scripts — thin wrappers over cluster_validator functions
  build_dataset.py      # Stream Dutch news → BERTopic → data/
  evaluate.py           # dspy.Evaluate + MLflow
  optimize.py           # BootstrapFewShot + MLflow
  optimize_gepa.py      # GEPA + MLflow
  optimize_finetune.py  # BootstrapFinetune (primary objective) + MLflow
  deploy_mlflow.py      # Log fine-tuned program to MLflow

outputs/                # Artifacts produced by pipeline runs (gitignored)
demo.ipynb              # Walkthrough notebook (eval + optimization)
Dockerfile              # Full-stack container with CUDA + SGLang
```

# LLM-Based Cluster Validation

An internal onboarding resource that walks through building, evaluating, optimizing, and fine-tuning a [DSPy](https://dspy.ai) program for cluster validation.

The concrete task is **intruder detection**: given six keywords where five belong to one semantic cluster and one is from a different cluster, an LLM must identify the intruder. This task is a vehicle for explaining DSPy concepts — the end goal is fine-tuning `ministral-3:4b` using traces generated by Claude Sonnet as teacher.

---

## What is the pipeline?

The entire pipeline is orchestrated by [Snakemake](https://snakemake.readthedocs.io) — see [Snakefile](Snakefile). Each rule imports the matching `run_*()` function from `pipeline/*.py` and runs it in-process.

```
build_dataset → data/raw_examples.json ─┬── evaluate         → outputs/eval_results.json
                                         ├── optimize        → outputs/program_optimized.json
                                         ├── optimize_gepa   → outputs/program_gepa.json
                                         └── optimize_finetune → finetuned_model/ + outputs/program_finetune.json
                                                                    └── deploy_mlflow → outputs/.deploy_marker
```

**Optimization order:** BootstrapFewShot → GEPA → BootstrapFinetune.
The first two establish a performance baseline and produce labeled traces; fine-tuning is the primary objective.

---

## What each stage does

### `evaluate` — baseline measurement

Runs the unoptimized `ClusterIntruderValidator` (a `dspy.ChainOfThought` over an `IntruderDetectionSignature`) across the dev set. `dspy.Evaluate` applies `intruder_exact_match` to every `(example, prediction)` pair and aggregates a percentage. MLflow autologging captures every LM call as a trace — inspect any failure in the MLflow UI.

### `optimize` — BootstrapFewShot

Improves the program **without touching model weights**. Runs the program on a small training set, keeps examples where the program answered correctly, and selects a subset as few-shot demonstrations to prepend to the prompt. Pure prompt engineering, automated.

### `optimize_gepa` — GEPA

**Rewrites the task instructions** themselves. Runs a reflection loop: collect failures → feed them to a reflection LM (Claude Sonnet via the `teacher` block in `config/dspy_config.yaml`) → score proposed instructions on the validation set → keep a Pareto frontier. Requires `ANTHROPIC_API_KEY`.

### `optimize_finetune` — BootstrapFinetune (primary objective)

**Updates model weights.** Teacher (Claude Sonnet) runs inference on the trainset; only examples where the teacher answers correctly become training data. Those `(prompt, completion)` pairs are passed to HuggingFace TRL for SFT of the student (`ministral-3:4b`), optionally with LoRA via PEFT. After training, the fine-tuned student no longer needs few-shot examples in the prompt.

Requires `ANTHROPIC_API_KEY`. The rule automatically stops SGLang before training (TRL needs the full GPU VRAM) and `pipeline/optimize_finetune.py` manages its own SGLang instances for the pre- and post-finetune evaluations.

### `deploy_mlflow`

Logs the fine-tuned program to MLflow using the `llm/v1/chat` schema so it can be served with `mlflow models serve`. Writes `outputs/.deploy_marker` containing the MLflow run ID.

## Setup

### Prerequisites

- Python 3.12
- [uv](https://docs.astral.sh/uv/) — for dependency management
- [SGLang](https://github.com/sgl-project/sglang) — serves `Ministral-4b-instruct` locally on GPU (the Snakefile starts and stops it for you via [scripts/sglang_manager.py](scripts/sglang_manager.py))
- An Anthropic API key (needed for `optimize_gepa` and `optimize_finetune`)

### Install

```bash
uv sync
```

### Environment variables

```bash
cp .env.example .env
# Edit .env and set your Anthropic API key:
# ANTHROPIC_API_KEY=sk-ant-...
```

> **Note — GCC version:** SGLang JIT-compiles FlashInfer CUDA kernels at first launch. CUDA requires GCC ≤ 12 as the host compiler. If your system GCC is newer (check with `gcc --version`), compilation will fail with an "unsupported GNU version" error. Install GCC 12 and register it via `update-alternatives` (the `CUDAHOSTCXX` env var alone is not sufficient — FlashInfer's ninja build does not pass it through to `nvcc`):
>
> ```bash
> sudo apt install gcc-12 g++-12
> sudo update-alternatives --install /usr/bin/gcc gcc /usr/bin/gcc-12 12
> sudo update-alternatives --install /usr/bin/g++ g++ /usr/bin/g++-12 12
> ```

## Running the pipeline

Inspect the DAG without running anything:

```bash
uv run snakemake --dry-run all
uv run snakemake --dag all | dot -Tpng > dag.png   # requires graphviz
```

Run the full pipeline end-to-end:

```bash
uv run snakemake all -j1
```

Run a single stage (Snakemake automatically builds any missing prerequisites):

```bash
uv run snakemake evaluate -j1
uv run snakemake optimize -j1
uv run snakemake optimize_gepa -j1
uv run snakemake optimize_finetune -j1
uv run snakemake deploy -j1
```

Override defaults per run — see [config/pipeline_config.yaml](config/pipeline_config.yaml) for all keys. Put the target **before** `--config` (Snakemake otherwise parses the target name as another key=value pair):

```bash
uv run snakemake optimize_gepa -j1 --config gepa_auto=heavy num_threads=8
```

Steps whose declared outputs already exist are skipped automatically. Force a re-run with `--forcerun <rule>` or wipe everything:

```bash
uv run snakemake clean
```

SGLang is started and stopped by the rules — do not launch it manually.

## Running with Docker

Build and run the full pipeline container (requires NVIDIA GPU):

```bash
docker build -t cluster-validator .

# Run any Snakemake target
docker run --gpus all \
    -e ANTHROPIC_API_KEY=$ANTHROPIC_API_KEY \
    -v $(pwd)/outputs:/app/outputs \
    cluster-validator \
    uv run snakemake evaluate -j1
```

Ports `30000` (SGLang) and `5000` (MLflow serve) are exposed.

---

## MLflow tracking

Every evaluation and optimization run is automatically logged via `mlflow.dspy.autolog()`.

```bash
uv run mlflow ui
# Open http://localhost:5000
```

Experiments:
- `cluster-validator-eval` — baseline evaluation runs
- `cluster-validator-optimize` — BootstrapFewShot and GEPA runs
- `cluster-validator-finetune` — BootstrapFinetune runs
- `cluster-validator-finetuned` — deployed models ready for serving

After `deploy`, serve the logged model (use the run ID written to `outputs/.deploy_marker`):

```bash
mlflow models serve -m "runs:/<run_id>/dspy_program" -p 6000
```

## Project structure

```
cluster_validator/      # Installable package — all reusable logic
  config.py             # configure_dspy(), configure_teacher_lm(), load_config()
  module.py             # IntruderDetectionSignature, ClusterIntruderValidator, find_intruder()
  metrics.py            # intruder_exact_match(), gepa_metric()
  data.py               # build_devset() and dataset splitting utilities

config/
  dspy_config.yaml      # Student (ministral) and teacher (Claude Sonnet) LM settings
  pipeline_config.yaml  # Orchestration defaults consumed by the Snakefile

data/                   # Input datasets (generated by build_dataset.py)
  topics.json
  raw_examples.json

pipeline/               # Executable scripts — thin wrappers over cluster_validator functions
  build_dataset.py      # Stream Dutch news → BERTopic → data/
  evaluate.py           # dspy.Evaluate + MLflow
  optimize.py           # BootstrapFewShot + MLflow
  optimize_gepa.py      # GEPA + MLflow
  optimize_finetune.py  # BootstrapFinetune (primary objective) + MLflow
  deploy_mlflow.py      # Log fine-tuned program to MLflow

scripts/
  sglang_manager.py     # SGLang lifecycle helpers used by the Snakefile

Snakefile               # DAG definition
outputs/                # Artifacts produced by pipeline runs (gitignored)
Dockerfile              # Full-stack container with CUDA + SGLang
```
